# i18n Design — Born in NoPo

**Date:** 2026-03-10

## Goal

Add bilingual support (English + Italian) to the blog, with locale-aware URLs, browser-based auto-detection, a language switcher, and translated UI strings.

## Requirements

- Both languages are prefixed: `/en/` and `/it/`
- Posts only appear in the language they're written in (no fallback)
- Browser locale is auto-detected on first visit; preference stored in `localStorage`
- A `EN / IT` toggle in the header lets users override at any time
- All UI strings translated (no hardcoded English labels)

## Routing

**`astro.config.mjs`** — add:
```js
i18n: {
  defaultLocale: 'en',
  locales: ['en', 'it'],
  routing: { prefixDefaultLocale: true },
}
```

**`src/pages/index.astro`** — bare HTML page with a `<script>` that:
1. Checks `localStorage.getItem('lang')`
2. Falls back to `navigator.language.startsWith('it') ? 'it' : 'en'`
3. Redirects to `/{lang}/`

All other page templates move under **`src/pages/[lang]/`**:
- `[lang]/index.astro`
- `[lang]/blog/index.astro`
- `[lang]/blog/[...slug].astro`
- `[lang]/[category].astro`
- `[lang]/about.astro`

Each uses `getStaticPaths()` returning `[{ params: { lang: 'en' } }, { params: { lang: 'it' } }]`.

## Content Structure

Posts move from `src/content/blog/` into locale subdirectories:

```
src/content/blog/
  en/
    first-post.md
    ...
  it/
    primo-post.md
    ...
```

Language is derived from the post `id` prefix (e.g. `en/first-post` → `en`). No new frontmatter field needed. The existing glob pattern (`**/*.{md,mdx}`) already handles subdirectories.

## Language Switcher

A `EN | IT` toggle added to the **header banner, top-right**, alongside the About link.

- Active locale is visually highlighted
- When the current page has a translation, both links are active
- When no translation exists, the other language link is dimmed and non-clickable
- Clicking a language saves the choice to `localStorage`

To detect whether a translation exists: on each page, query posts for the same slug in the other locale.

## UI Strings

**`src/i18n/translations.ts`** — a simple lookup object, no external library:

```ts
export const TRANSLATIONS = {
  en: {
    recentPosts: 'Recent Posts',
    allPosts: 'All posts →',
    allPostsLabel: 'All Posts',
    noPosts: 'No posts yet — check back soon.',
    // ... etc
  },
  it: {
    recentPosts: 'Post recenti',
    allPosts: 'Tutti i post →',
    allPostsLabel: 'Tutti i post',
    noPosts: 'Nessun post ancora — torna presto.',
    // ... etc
  },
}

export type Lang = keyof typeof TRANSLATIONS;
export const t = (lang: Lang) => TRANSLATIONS[lang];
```

Each page receives `lang` from `Astro.params` and passes `t(lang)` to components that need it.

Category display labels are also included in `TRANSLATIONS` (slugs stay the same across locales).
